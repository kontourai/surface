# Surface Agent Guidance

Surface is the canonical source for this repo. Treat generated agent bundles, session prompts, and Flow Agents runtime artifacts as local coordination state unless a tracked source file explicitly says otherwise.

## Source Of Truth

- Product and architecture context lives in `CONTEXT.md`, `README.md`, and `docs/`.
- Public package entry points are `src/index.ts`, `bin/surface.mjs`, package `exports`, and package `bin`.
- Source docs live under `docs/`. `docs-site/` is generated by `npm run docs:build` as a curated public subset and should not be edited directly.
- Generated/local artifacts such as `dist/`, `test-results/`, `playwright-report/`, `.surface/`, and `.flow-agents/` should be regenerated or inspected for evidence, not treated as product source.

## Development Rules

- Keep Surface behavior stable unless the task explicitly asks for runtime changes: validation, trust derivation, report projection, adapters, extension registry, CLI, and Console routes are compatibility-sensitive.
- Prefer small edits that match existing TypeScript and docs style.
- Do not add empty placeholder directories. Current adapter docs and examples are `docs/adapters.md` and `examples/external-adapter/`.
- Do not hand-edit generated Pages output in `docs-site/`.
- Do not revert unrelated user or agent changes.

## Useful Commands

- `npm run build`
- `npm test`
- `npm run docs:build`
- `npm run test:browser`
- `npm run verify`
